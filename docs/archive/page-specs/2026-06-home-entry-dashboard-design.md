# 首页年度进店结构看板需求设计

> **已迁移**：生效版本见 [首页年度进店结构看板设计](../../17-home-entry-dashboard-design.md)（含业务范围方案 A 与实现锚点）。本文档仅作历史追溯。

> 状态：归档  
> 更新日期：2026-06-09  
> 适用范围：首页新增「进店」大模块

## 1. 背景与目标

首页需要新增一个独立的「进店」大模块，用于观察部门全年进店节奏、直营 / 加盟结构、新店 / 老店结构、店态结构和区域分布。

该模块只参考钉钉截图表达的业务方向，不沿用钉钉表单的拆分方式和展示逻辑。本系统中不再拆成「新店项目量」和「老店项目量」两个独立模块，而是整合为一个可下钻的年度进店结构看板。

模块核心回答：

- 今年每个月进店量是多少。
- 每个月新店和老店分别多少。
- 新店 / 老店内部直营和加盟分别占多少。
- 某个月的进店压力主要来自哪些店态。
- 某个月的进店项目集中在哪些省份。
- 年度总览、月份下钻和区域下钻的数据是否完全一致。

## 2. 已确认统计口径

### 2.1 进店项目范围

进店量按「进店事实」统计，而不是按当前是否仍在推进统计。

计入规则：

- 进店日期落在选定年份或选定月份内。
- 项目属于系统有效业务范围。
- 已闭环项目仍然计入，因为闭环不改变历史进店事实。

排除规则：

- 暂停项目不计入。
- 取消项目不计入。
- 无法识别为直营或加盟范围的项目暂不进入主图，可在数据说明中提示「未归类项目」数量。

### 2.2 时间口径

优先使用标准字段 `startDate`，对应钉钉「启动时间 / 启动日期」等进店日期字段。

年度筛选：

- 默认展示当前看板年份。
- 只统计 `startDate` 落在该自然年的项目。
- 1-12 月必须完整展示，即使某个月为 0 也保留月份位置。

月份筛选：

- 点击某个月后，只统计该月 1 日至该月最后一天内进店的项目。
- 月份详情、区域分布、hover 明细和项目下钻必须使用同一批月份项目。

不建议在该模块中使用 `updatedAt` 回退统计进店量。若 `startDate` 缺失，应暴露为数据质量问题，而不是用更新时间替代进店事实。

### 2.3 新店 / 老店口径

优先读取能够表达新老店的标准字段或原始字段，例如「店铺性质」。

建议分类：

| 分类 | 判定说明 |
| --- | --- |
| 新店 | 字段明确为新店，或业务字段可稳定识别为新开项目 |
| 老店 | 字段明确为老店、改造、翻新、重装等存量门店项目 |
| 未识别 | 字段为空或无法稳定归类，不进入新店 / 老店双柱主图，可在数据说明中提示 |

主图每个月固定展示两根柱子：

- 新店
- 老店

### 2.4 直营 / 加盟口径

直营 / 加盟沿用现有 profile scope 口径：

- 直营：`组别` 或清洗后的 scope 命中直营。
- 加盟：`组别` 或清洗后的 scope 命中加盟。

不得用店态、项目名称或负责人反推直营 / 加盟。

### 2.5 店态口径

店态使用原始或清洗后的「店态」字段，例如常规店、下沉店、高标店、旗舰店、黑标店、睡眠店、儿童店、维莎店等。

店态不作为主图柱体颜色的主要维度。主图颜色优先表达直营 / 加盟，店态进入 hover、月份详情和排行图展示。

### 2.6 省份口径

省份使用标准字段 `province`，并复用现有省份显示规范。

区域子模块按省份聚合，不默认合并大区。若后续需要华东、华南等大区视角，另行定义，不在本期需求内。

## 3. 首页模块信息架构

模块名称建议为「年度进店结构」。

整体结构：

```text
年度进店结构

年度总览
1月  2月  3月  ...  12月
新 老  新 老  新 老      新 老

月份详情
直营新店 / 直营老店 / 加盟新店 / 加盟老店
店态排行
店态 × 新老店 × 直营加盟结构

区域分布
月份选择
省份总量排行
每省新店 / 老店拆分
```

## 4. 年度总览主图

### 4.1 图表形态

采用 1-12 月分组柱状图。

每个月固定两根柱子：

- 新店柱
- 老店柱

每根柱子内部按直营 / 加盟做堆叠：

- 直营使用一种稳定颜色。
- 加盟使用另一种稳定颜色。
- 同一颜色在所有月份和下钻面板中保持一致。

主图不按店态拆颜色，避免颜色过多导致阅读困难。

### 4.2 主图展示内容

每个月需要展示：

- 当月新店总量。
- 当月老店总量。
- 新店中直营 / 加盟数量。
- 老店中直营 / 加盟数量。

年度总量需要能由 12 个月求和得到：

```text
年度进店总量 = 所有月份新店总量 + 所有月份老店总量
```

若存在未识别新老店项目，应单独显示为数据提示，不混入新店 / 老店柱。

### 4.3 hover 明细

鼠标悬浮到某个月某根柱子时，展示该月该类型的细分明细。

示例：

```text
2026年 3月 · 新店进店 18 项

直营 7 项，占 39%
加盟 11 项，占 61%

店态结构
常规店 8
下沉店 4
高标店 3
旗舰店 2
其他 1

省份 Top
浙江省 4
江苏省 3
四川省 2
```

hover 明细必须来自当前柱子的同一批项目，不得重新使用其他筛选逻辑计算。

## 5. 月份下钻交互

### 5.1 交互方式

点击某个月后，在模块内部展开月份详情面板。

不建议使用弹窗作为主交互，因为首页看板需要连续比较多个月份；模块内展开可以减少打断。

点击行为：

- 点击月份标签或该月任意柱子，选中该月份。
- 默认选中有数据的最近月份；若全年无数据，则展示空状态。
- 再次点击其他月份时，详情面板直接切换到新月份。

### 5.2 月份详情内容

月份详情按「直营 / 加盟 × 新店 / 老店」做四象限摘要：

```text
选中：2026年 3月

直营新店 7 项
直营老店 5 项
加盟新店 11 项
加盟老店 4 项
```

每个象限展示：

- 项目数量。
- 占当月总进店比例。
- 主要店态 Top 3。
- 主要省份 Top 3。

### 5.3 店态详情

月份详情中增加店态排行。

推荐结构：

```text
店态排行
常规店  12 项
下沉店   8 项
高标店   4 项
旗舰店   3 项
```

店态排行点击后可切换右侧明细或进入项目列表筛选：

- 月份 = 当前选中月份
- 店态 = 当前店态
- 进店范围 = 有效进店项目

本期可以先实现为展示型排行，保留点击项目明细的后续扩展口。

### 5.4 月份详情的数据一致性

月份详情必须满足：

```text
当月总量
= 直营新店 + 直营老店 + 加盟新店 + 加盟老店
+ 未识别但被显式提示的项目
```

主图中该月新店柱和老店柱的合计，必须等于月份详情中的新店 / 老店合计。

## 6. 区域分布子模块

### 6.1 子模块定位

区域分布属于「进店」大模块的子模块，用于回答某个月进店集中在哪些省份。

本期不做地图，优先采用省份条形排行。原因：

- 首页 2K 桌面端中，条形排行更容易读数。
- 省份数量通常不多，可以完整展示。
- 条形排行更适合继续拆新店 / 老店。

### 6.2 展示方式

区域子模块顶部保留月份选择，与主图选中月份联动。

主体展示所有有进店项目的省份：

```text
2026年 3月区域分布

浙江省  6 项    新店 4 / 老店 2
江苏省  5 项    新店 2 / 老店 3
四川省  3 项    新店 3 / 老店 0
山东省  2 项    新店 1 / 老店 1
```

每个省份展示：

- 省份总进店量。
- 新店数量。
- 老店数量。
- 可选：直营 / 加盟小标记或 hover 明细。

如果某个月省份超过 15 个，可使用紧凑双列布局；不默认折叠隐藏。

### 6.3 区域下钻

点击某个省份时，后续可进入项目明细筛选：

- 年份 = 当前年份
- 月份 = 当前月份
- 省份 = 当前省份
- 排除暂停 / 取消
- 闭环项目保留

本期若只做看板，不做项目列表跳转，也要在结构上保留筛选参数，便于后续接入。

## 7. 数据结构建议

## 8. 正确性约束

实现时必须保证以下一致性：

1. 年度总量等于 12 个月总量之和。
2. 每个月总量等于该月新店总量 + 老店总量 + 显式未识别数量。
3. 每个月新店总量等于新店柱中的直营 + 加盟 + 显式未归类 scope。
4. 每个月老店总量等于老店柱中的直营 + 加盟 + 显式未归类 scope。
5. 月份详情四象限合计必须和主图该月合计一致。
6. 区域子模块省份总量合计必须和当前月份总量一致。
7. 店态排行总量合计必须和当前月份总量一致。
8. 暂停 / 取消项目在所有层级都排除。
9. 闭环项目在所有层级都保留。
10. hover、月份详情、区域排行和后续项目明细入口必须共享同一批筛选结果。

## 9. 首页布局建议

本项目首页按 2K 桌面端体验设计，不考虑移动端适配。

建议模块布局：

```text
┌──────────────────────────────────────────────────────────────┐
│ 年度进店结构                                                  │
│ 年份 / 数据说明 / 总进店量 / 新店 / 老店 / 直营 / 加盟          │
├──────────────────────────────────────────────────────────────┤
│ 1-12月双柱堆叠图                                              │
├───────────────────────────────┬──────────────────────────────┤
│ 月份四象限详情                  │ 店态排行 / 结构摘要             │
├───────────────────────────────┴──────────────────────────────┤
│ 区域分布：省份排行 + 新店/老店拆分                              │
└──────────────────────────────────────────────────────────────┘
```

视觉层级：

- 主图占模块最大面积。
- 月份详情默认收起或显示默认选中月份，避免首页过长。
- 区域分布放在详情下方，跟随月份选择联动。
- 不使用过多卡片嵌套，避免首页显得碎。

## 10. 空状态与数据质量提示

需要明确展示以下情况：

- 当年无进店数据：显示空状态，不隐藏模块。
- 某个月无数据：主图该月保留 0 值位置，详情显示空状态。
- `startDate` 缺失：在数据说明中显示缺失数量，不使用 `updatedAt` 替代。
- 新老店无法识别：显示未识别数量，不强行归入新店或老店。
- 直营 / 加盟无法识别：显示未归类数量，不强行归入直营或加盟。
- 省份缺失：显示为「省份未填写」，并计入当前月份总量。

## 11. 测试建议

至少覆盖以下测试：

- 暂停项目不计入年度和月份进店量。
- 取消项目不计入年度和月份进店量。
- 已闭环项目仍计入年度和月份进店量。
- 同一年 12 个月聚合总数等于年度总量。
- 选中某个月后，主图、月份详情和区域分布总数一致。
- 新店柱内部直营 + 加盟等于新店总量。
- 老店柱内部直营 + 加盟等于老店总量。
- 省份排行合计等于当前月份总量。
- 店态排行合计等于当前月份总量。
- 缺失 `startDate` 的项目不进入进店量，并进入数据质量提示。

## 12. 暂不纳入本期

以下能力暂不纳入本期首页模块：

- 地图可视化。
- 大区维度汇总。
- 进店量与人员负载、延期风险、排班的联动研判。
- AI 自动总结。
- 项目明细弹窗或完整列表改造。
- 移动端布局。

这些能力可以在本模块数据结构稳定后继续扩展。
