# 负责人复盘页 UI 规格

> 归档说明：页面 UI 规格从根 README 迁出，避免启动文档承载过多交互细节。

负责人复盘页入口是 `#owner-review`，页面骨架在 `public/index.html`，主要渲染逻辑在 `public/app.js`，样式在 `public/styles.css`。它是负责人视角的当前团队负载工作台，不是月度复盘页；默认按桌面端宽屏体验维护。页面数据来自 `GET /api/team-responsibility-review`，前端只负责展示和交互，不默认改钉钉字段含义或后端责任判定。

- 顶部区域负责选择负责人、展示团队负载摘要和「跨团队借调」开关；借调开关只影响当前可见复盘数据，不改变原始数据。
- 团队架构区展示负责人下属直营组、成员和当前平面负载。成员卡片可打开成员负载弹窗；不在岗占位成员按空载样式展示，不计入真实当前负载。
- 今日决策摘要只展示 4 个可点击卡片：`需立即干预`、`可调配人手`、`高风险组`、`数据待核对`。点击后打开 `ownerReviewDecisionModal` 弹窗展示对应明细，不把明细长列表直接铺在页面下方。
- `团队负载工作台` 是一个整合模块：左侧 `组间对比` 矩阵用于选择直营组，右侧 `选中组负载详情` 就地展示该组成员、KPI、当前平面、空载成员和异常。点击组间对比行会清空搜索与筛选，并只刷新右侧详情。
- 工作台不再渲染所有组的堆叠折叠详情；也不保留「展开有负载组」「展开异常组」「收起全部」「标准/紧凑」这类按钮。布局默认就是标准密度，后续不要重新引入密度切换，除非重新确认产品口径。
- 负载筛选使用自定义下拉外壳包住原生 `#ownerReviewLoadFilter`，选项为 `全部成员`、`有当前平面`、`空闲成员`、`偏高/过载`、`数据待核查`；搜索框支持按员工、直营组、项目、门店定位。
- 负载口径：只有当前未完成的平面方案计入当前负载；平面完成进入历史完成，摆场只做进行/完成统计，关联记录仅作为复盘证据。负载等级为 `空闲=0`、`正常=1-2`、`偏高=3-4`、`过载>=5`。
- 成员明细弹窗会分组展示当前/已完成的平面、摆场、关联项目，并支持继续打开项目详情；成员弹窗提供复盘摘要复制，方便运营沟通。
- 修改本页时同步检查 `tests/brand-ui.test.mjs` 和 `tests/publicAppBehavior.test.mjs`。这两类测试分别守住页面骨架、关键 CSS/DOM 入口、自定义下拉、决策弹窗、标准密度布局、组选中详情和成员弹窗行为。
