# 项目明细改版设计

日期：2026-05-29

## 背景

当前项目明细页直接把钉钉原始字段动态铺成一张 50+ 列的大宽表。负责人需要横向和纵向反复滚动才能看到关键项目状态，页面显得笨重，也不适合日常项目扫查。

本次先改项目明细页的信息架构和前端呈现，不改变数据同步、指标口径和后端 API。

## 目标

项目明细默认第一屏从“大宽表”改为“项目状态扫描工作台”。

默认可见内容服务于负责人每天快速判断：

- 哪些项目需要处理。
- 项目当前卡在哪个阶段。
- 谁负责。
- 是否延期或有风险。
- 点击后如何继续深查。

## 交互模型

页面保留独立项目明细页，同时支持单项目弹窗/抽屉式详情。

一级展示：

- 项目名称
- 地区
- 店态 / 业态
- 负责人
- 当前项目状态
- 关键节点日期
- 延期 / 风险提示
- 打开详情入口

二级展示：

- 单项目详情抽屉或弹窗
- 硬装 / 软装节点分组
- 人员信息
- 关键节点日期和完成情况
- 常用业务字段

三级展示：

- 钉钉原始字段
- 附件和链接
- 长文本备注
- 对账和追溯字段

三级内容默认折叠，只在负责人需要查具体数据时展开。

## 明细页结构

项目明细页由两块组成：

1. 页内筛选区：搜索、店态、业态、状态等现有筛选逻辑保留，但从页面顶部大筛选框改为项目明细内部的轻量折叠筛选。
2. 项目清单区：默认不再按全部字段横向铺开，而是以高密度项目列表展示关键字段。

点击项目后打开二级弹窗。弹窗内按“项目概况、当前推进、日期与提醒、管理边界、人员协作、资料链接”折叠展示。项目明细页不展示“钉钉原始字段”模块，也不再提供钉钉原始大表入口。

## 设计原则

- 默认页不横向滚动。
- 项目名称和详情入口始终容易找到。
- 常用字段一级可见，低频字段折叠。
- 每个项目可以从列表快速打开二级详情弹窗。
- 弹窗和未来独立项目页使用同一套字段分组。
- 视觉风格保持源氏木语当前绿色品牌，但减少厚重边框和大块卡片堆叠。

## 验收标准

- 项目明细第一屏不再依赖底部横向滚动查看核心信息。
- 负责人可以在列表中快速看到项目、负责人、店态、状态、下一提醒和风险。
- 点击任意项目可以打开二级详情弹窗。
- 详情弹窗中常用信息默认可见，低频字段折叠。
- 项目明细页不展示钉钉原始字段大表。
- 搜索和现有筛选逻辑继续可用。
- 页面在桌面宽屏和较窄屏幕下不出现明显文本重叠。
