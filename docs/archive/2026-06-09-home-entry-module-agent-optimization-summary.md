# 首页进店模块多 Agent 改造方案汇总

> 状态：已归档（多 Agent 方案汇总稿，已被 V3 取代）  
> 日期：2026-06-09  
> 归档日期：2026-06-10  
> 关联文档：[V3 落地稿](./2026-06-10-home-entry-structure-v3-plan.md)  
> 范围：仅优化首页截图中的「年度节奏 / 年度进店结构」模块，不扩展到其他首页模块。  
> 目标：保留现有统计口径和已有数据资产，把前端从“数据堆叠展示”升级为“经营判断 + 交互下钻入口”。

## 1. 背景判断

当前模块的数据完成度已经较高，前端主要问题不是缺字段，而是信息组织、视觉层级和交互状态没有形成一个统一产品。

从截图和当前实现看，模块已经具备：

- 年度总量、新店、老店、直营、加盟汇总。
- 全年 1-12 月趋势。
- 当前月份聚焦。
- 店态排行、省份排行。
- 按月份打开项目明细入口。
- 年份、月份、全年、季度、月份按钮等交互。

但这些内容目前像被平铺在同一层级，导致用户需要自己重新读图、算比例、判断异常，再决定下一步看哪里。

本次改造应围绕三个核心问题：

1. 本月进店是否异常。
2. 当前结构是否健康。
3. 应该点击哪些项目继续追踪。

## 2. 多 Agent 视角结论

### 2.1 信息架构 Agent

核心判断：数据重复过多，尤其是 `74项`、`2026年5月`、`新店占比`、`直营/加盟` 在多个区域重复出现，但没有说明哪些是年度口径、哪些是当前月口径。

建议：

- 把 `当前聚焦：2026年5月 / 74项` 作为唯一主状态。
- 年度数据降级为背景参照。
- 顶部只保留年度总览和结构概览，不再重复当前月。
- 右侧详情改为“当前选择解释器”，不再重复 KPI 卡。
- 排行图改为“贡献来源和下钻入口”，而不是静态排行。

### 2.2 视觉设计 Agent

核心判断：现在的界面像很多小卡片拼在一起，按钮、边框、图表、提示块的完成度不统一。

建议：

- 采用“运营分析驾驶舱”的克制风格，高密度但强对齐。
- 减少每个小块的重边框，更多用分区、留白和层级表达。
- 季度按钮改成标准 segmented control。
- 月份按钮收进图表横坐标，横坐标月份本身承担点击和数值展示，不再额外放一排月份卡。
- 右侧详情面板要有主标题、结论和结构摘要；明细入口放到横坐标月份按钮。
- 图表当前月高亮用竖向背景带，而不只是柱体描边。

### 2.3 交互体验 Agent

核心判断：当前存在多个控件都能改变视图，但用户不清楚现在到底处在全年、季度还是单月口径。

建议统一状态机：

```text
year: number
scopeType: annual | quarter | month
quarter: Q1 | Q2 | Q3 | Q4 | null
month: 1-12 | null
selectedSegment: null | newStore | oldStore | direct | franchise | province | storeStatus
```

关键规则：

- 全年、季度、月份互斥。
- 年份切换后默认选中该年最新有效月份。
- 点击柱状图只改变当前聚焦，不直接弹出项目明细。
- 点击横坐标月份按钮才进入该月项目清单。
- 点击排行项生成筛选标签，例如 `广东省 x`、`常规店 x`。
- 最多允许一个二级筛选，避免首页变成复杂报表工具。

### 2.4 业务洞察 Agent

核心判断：这个模块应该先讲经营判断，再给数据证据。

基于截图示例：

- 2026 年已记录进店 `374项`，覆盖 `6/12` 个有效月份。
- 3 月峰值 `94项`，5 月 `74项`，属于高位回落。
- 年度新店占比 `71%`，5 月新店占比约 `68%`，新店是核心驱动力。
- 年度直营 `47%`、加盟 `53%`，5 月直营/加盟接近均衡。
- 7-12 月不能直接显示成经营为 0，需要标识为未来月份或暂无数据。

建议让模块形成一句判断加三层证据：

```text
经营判断
-> 年度节奏证据
-> 当前月结构证据
-> 店态/省份来源证据
```

## 3. 当前实现锚点

当前前端核心在：

- `public/dashboard/annual-entry-structure.mjs`
- `public/styles/pages/details.css`
- `public/pages/overview.mjs` 中通过 `mountAnnualEntryStructure` 挂载。

当前后端数据核心在：

- `src/backend/metrics/buildAnnualEntryStructure.mjs`
- `src/backend/metrics/entryScope.mjs`

当前实现已经有较完整状态：

```javascript
selectedYear
selectedMonth
selectedQuarter
contextMode
modal
loading
error
```

但现有 UI 结构存在几个关键问题：

- `renderSummaryTotals` 输出 5 个年度 KPI 卡，视觉权重过高。
- `renderSignalStrip` 再次输出当前口径、有数据月份、峰值月份、新店占比，和顶部/右侧重复。
- `renderContextInspector` 再次输出口径总量、月均、新店占比、直营占比，和其他区域重复。
- `renderMonthDrillStrip` 让月份和主图横坐标重复，且 0 数据月份和有效月份区分不足。
- `renderCharts` 中点击主图月份会直接打开项目 modal，这让“读图选择”和“查看明细”混成一个动作。
- CSS 中大量卡片边框、浅色背景、相似字号让主次层级不明显。

## 4. 推荐总方案

推荐采用：

**经营判断区 + 年度主图 + 当前选择解释器 + 归因排行区 + 项目清单入口**

布局建议：

```text
┌────────────────────────────────────────────────────────────────────┐
│ 年度进店结构                                      年份 / 当前聚焦   │
├────────────────────────────────────────────────────────────────────┤
│ 经营判断区：年度总览 + 当前月状态 + 结构判断                       │
├────────────────────────────────────────────────────────────────────┤
│ 时间范围 segmented control：全年 / Q1 / Q2 / Q3 / Q4               │
├───────────────────────────────────────────┬────────────────────────┤
│ 1-12 月年度节奏主图                         │ 当前选择解释器          │
│ 横坐标月份按钮：月份 + 直营/加盟数值 + 明细入口 │ 结论 / 结构说明          │
├───────────────────────────────┬───────────┴────────────────────────┤
│ 店态贡献排行                   │ 省份贡献排行                        │
└───────────────────────────────┴────────────────────────────────────┘
```

2K 桌面端建议：

- 外边距：24px。
- 模块间距：12-16px。
- 主图宽度：约 68%-72%。
- 右侧解释器宽度：约 28%-32%。
- 主图高度：320-360px。
- 底部排行高度统一，不出现突兀原生滚动条。

## 5. 数据取舍方案

### 5.1 一级常驻信息

| 信息 | 展示方式 | 说明 |
| --- | --- | --- |
| 年度总进店 | 顶部经营判断区 | 年度背景，不和当前月抢主角 |
| 当前聚焦月份和数量 | 顶部状态 + 右侧解释器标题 | 只作为唯一主状态出现 |
| 峰值月份 | 经营判断区或主图标记 | 用于判断节奏高低 |
| 新店/老店结构 | 结构比例条 | 年度与当前月对比 |
| 直营/加盟结构 | 结构比例条 | 年度与当前月对比 |
| 主图 1-12 月 | 中部主视觉 | 讲时间节奏 |
| 店态/省份排行 | 底部归因区 | 讲贡献来源 |
| 项目明细入口 | 横坐标月份按钮 | 打开对应月份清单 |

### 5.2 合并或降级信息

| 当前信息 | 建议处理 |
| --- | --- |
| `当前口径 74项`、右侧 `口径总量 74项`、按钮 `74项` | 合并为一个当前聚焦状态 |
| 顶部 `新店 267 71%` 和信号条 `新店占比 71%` | 顶部保留年度，信号条删除或改为当前月对比 |
| `月均节奏 74项/月` | 改为较活跃月均差异，否则和当前月重复 |
| `有数据月份 6/12` | 降级为数据完整度说明或节奏辅助标签 |
| 7-12 月 `0` | 标识为未来月份或暂无数据，不按普通 0 值强调 |
| Top8 外排行 | 合并为其他，详细列表进入清单或 tooltip |

### 5.3 不建议继续平铺的信息

- 不在顶部同时铺开 5 张大 KPI 卡。
- 不在右侧重复年度 KPI。
- 不把月份下拉、季度按钮、月份卡做成三个看似独立的筛选系统。
- 不让图表点击直接打开明细弹窗。

## 6. 模块结构改造

### 6.1 顶部：经营判断区

目标：用户 5 秒内知道当前状态。

建议内容：

```text
2026 年进店 374 项
当前聚焦 5 月 74 项，较 3 月峰值 94 项回落 20 项
新店占比 68%，直营/加盟 36/38，结构基本均衡
```

建议组件：

- 主指标：年度进店、当前聚焦月。
- 两条结构比例条：新店/老店、直营/加盟。
- 一句经营判断：高位回落、结构均衡、需要观察 6 月是否完整。
- 一个主入口：查看当前范围项目。

### 6.2 时间范围控制

将 `全年 / Q1 / Q2 / Q3 / Q4` 改为标准 segmented control。

交互规则：

- 默认选中当前聚焦月份对应状态。
- 点击全年后，右侧解释器切到全年。
- 点击季度后，主图仍显示 12 个月，但季度范围高亮。
- 点击月份后，季度状态自动退出。
- 0 数据或未来季度可显示弱化态，但不隐藏。

### 6.3 主图：年度节奏

主图保留“每月两根柱”的业务读法：每个月拆成直营、加盟两根柱，每根柱内部再按新店/老店堆叠：

```text
每月两根柱
直营柱 = 直营新店 + 直营老店
加盟柱 = 加盟新店 + 加盟老店
横坐标标签 = 月份 + 直营数 + 加盟数
```

这样用户读图路径更简单：

- 横向看月份节奏。
- 左右两根柱直接比较直营/加盟。
- 柱内颜色段看新店/老店来源。
- 横坐标下方数值和月份严格对齐。
- 高亮带看当前选择。

主图必须补齐：

- 当前月竖向高亮带。
- 峰值月标记。
- 活跃月均参考线。
- 0 数据月份弱化。
- hover tooltip。
- 点击柱子只切换右侧解释器；点击横坐标月份按钮打开对应月份明细。

### 6.4 横坐标月份按钮

取消图表下方独立的 12 个等权月份卡，把月份选择并入 ECharts 横坐标标签。

规则：

- 横坐标标签显示月份、直营数、加盟数。
- 当前月使用绿色描边和浅绿底。
- 未来月份或 0 数据月份使用灰色弱化，不和有效月份同权。
- hover 只显示预览 tooltip，不改变全局状态。
- click 才改变 `scopeType = month`。

### 6.5 右侧：当前选择解释器

右侧不再叫“当前聚焦月份”的小看板，也不承担明细按钮入口，而是解释当前选择和结构判断。

默认单月内容：

```text
2026年5月
进店 74 项

判断：
5月低于3月峰值和4月，但仍处于全年高位。

结构：
新店 50 项，占 68%
直营 36 项，加盟 38 项，渠道基本均衡

行动：
如需查看项目，点击主图横坐标下方的 5月 按钮。
```

季度/全年内容：

- 展示范围总量。
- 展示该范围峰值月。
- 展示结构占比。
- 展示主要贡献省份/店态。
- 右侧只说明当前范围，不生成明细入口。

如果点击排行或结构筛选：

- 标题改成具体对象，例如 `2026年5月 · 广东省`。
- 横坐标月份按钮仍负责打开对应月份清单，排行筛选只改变右侧解释上下文。
- 显示筛选标签，支持清除。

### 6.6 底部：归因排行区

底部店态排行和省份排行不再只是图表，而是“贡献来源入口”。

建议：

- 标题写清当前口径，例如 `5月店态贡献`、`5月省份贡献`。
- Top 5 常驻，其他合并。
- 每行或每条 hover 显示占比和拆分。
- 点击排行项生成二级筛选，右侧解释器同步变化。
- 只允许一个二级筛选，避免状态爆炸。

## 7. 按钮和状态规范

### 7.1 年份选择

- 保留为 select。
- 切换时保留旧数据可读，标题区域显示刷新中。
- 失败时保留旧数据，并展示重试提示。

### 7.2 月份选择

- 不建议同时强化顶部月份下拉和下方月份卡。
- 若保留顶部月份下拉，应定位为“快速定位当前聚焦”，和图表柱子点击、横坐标月份点击共享同一个状态。
- 不再让顶部月份选择看起来像另一个独立筛选系统。

### 7.3 季度 segmented control

状态：

- 默认：浅底、弱边框。
- hover：浅绿底、边框加深。
- active：深绿底、白字。
- disabled/future：灰字，低透明度，不抢视觉。

### 7.4 月份明细按钮

明细入口不放在右侧解释器里，放在主图每个月横坐标下方的月份按钮里：

```text
1月  直22 盟37
2月  直16 盟16
5月  直36 盟38
```

按钮视觉：

- 月份胶囊使用主色描边。
- 直营数使用绿色，加盟数使用蓝色。
- 当前聚焦月份使用浅绿底。
- hover 有明确可点击反馈。

### 7.5 弹窗与抽屉

当前实现使用 modal 展示项目明细。后续更推荐右侧抽屉或跳转清单页。

第一阶段可以保留 modal，但必须调整触发：

- 点击柱子：只切换聚焦。
- 点击横坐标月份按钮：打开该月项目清单。
- 点击排行项：只进入二级筛选并更新右侧解释器，不立即弹清单。

## 8. 业务洞察文案模板

### 8.1 年度总览

```text
2026年已记录进店 374 项，覆盖 6 个有效月份。3月达到峰值 94 项，4-5月仍处高位但较峰值回落。年度进店主要由新店贡献，新店占比 71%；直营与加盟结构接近均衡，加盟略高。
```

### 8.2 当前月

```text
当前查看 2026年5月，进店 74 项。本月低于4月 87 项和3月峰值 94 项，但仍高于多数月份，属于高位回落。本月新店 50 项，占 68%；直营 36 项、加盟 38 项，渠道结构较均衡。
```

### 8.3 数据完整度

```text
当前年度已覆盖 6 个有效月份；未来月份暂不纳入趋势异常判断。6月如仍处进行中，不参与完整月份环比判断。
```

### 8.4 排行解释

```text
店态和省份排行用于识别贡献来源。优先查看贡献最高的店态或省份是否同时集中在新店、加盟或某个经营结构中。
```

## 9. 健康与异常规则

第一阶段仅做描述型判断，不写成强业务结论。

| 类型 | 建议规则 | 文案口径 |
| --- | --- | --- |
| 总量关注 | 当前月较上月下降 15%-20% | 高位回落，需观察 |
| 总量异常 | 连续 2 个完整月份下降，且低于有效月份均值 | 疑似低位，需解释 |
| 新店驱动 | 新店占比 60%-75% | 新店驱动明显 |
| 新店偏高 | 新店占比高于 75% | 老店贡献偏低，需关注 |
| 渠道均衡 | 直营/加盟差距小于 10 个百分点 | 渠道结构均衡 |
| 渠道依赖 | 单边超过 65% | 渠道贡献集中 |
| 区域集中 | Top 3 省份超过 60% | 区域集中度较高 |
| 数据不完整 | 未来月或进行中月份 | 不纳入异常判断 |

注意：如果没有目标值、同比、确认过的运营阈值，不使用“达标/未达标”“好/差”等强判断。

## 10. 工程落地建议

### 10.1 保持后端口径不变

第一阶段不改后端数据口径，继续使用现有 payload：

- `year`
- `defaultMonth`
- `totals`
- `months`
- `months[].newStore`
- `months[].oldStore`
- `months[].quadrants`
- `months[].storeStatuses`
- `months[].provinces`
- `months[].projects`

### 10.2 前端新增 view model

建议在 `public/dashboard/annual-entry-structure.mjs` 内先增加 view model 层，避免渲染模板里直接拼判断。

建议拆分：

```text
buildEntryStructureViewModel(payload, state)
renderEntryDecisionBand(viewModel)
renderEntryTimeControl(viewModel, state)
renderEntryMainChart(viewModel, state)
renderEntryInspector(viewModel, state)
renderEntryRankingPanels(viewModel, state)
buildEntryProjectFilter(state, source)
```

### 10.3 状态调整

当前的 `contextMode` 可以继续使用，但建议逐步改成更明确的：

```text
scopeType: annual | quarter | month
selectedQuarter
selectedMonth
selectedSegment
selectedProvince
selectedStoreStatus
```

短期可以用适配层映射：

```text
contextMode = year    -> scopeType = annual
contextMode = quarter -> scopeType = quarter
contextMode = month   -> scopeType = month
```

### 10.4 图表点击行为调整

当前主图需要区分“读图聚焦”和“打开清单”。建议改为：

```text
点击柱状图:
  scopeType = month
  selectedMonth = clickedMonth
  modal.open = false
  inspector 更新

点击横坐标月份按钮:
  scopeType = month
  selectedMonth = clickedMonth
  modal.open = true
  filter = all
```

这会显著改善交互观感。

### 10.5 CSS 优化方向

`public/styles/pages/details.css` 中当前进店模块样式应按组件重组：

- 标题与筛选区。
- 经营判断区。
- segmented control。
- 主图与横坐标月份按钮。
- 右侧解释器。
- 排行区。
- 明细 modal/drawer。

视觉上减少重复边框，让主图和右侧解释器成为第一视觉中心。

## 11. 分阶段实施

### P0：完成度从 30%-40% 拉到可用

- 合并重复指标。
- 重做顶部经营判断区。
- 重做季度和月份按钮体系。
- 主图点击不再直接弹窗。
- 右侧面板改为当前选择解释器。
- 0 数据/未来月份弱化。
- 横坐标月份按钮承载明细入口。

### P1：增强运营分析感

- 主图增加均值线、峰值标记、当前月背景高亮。
- 店态/省份排行支持点击生成筛选标签。
- 右侧解释器支持结构、排行筛选后的上下文文案。
- tooltip 补齐月度结构、占比、贡献来源。

### P2：补充经营闭环

- 项目明细从 modal 升级为抽屉或清单页。
- 明细入口携带筛选条件。
- 接入环比、近 3 月均值、集中度、结构偏移。
- 预留待跟进队列，但不在本阶段强做。

## 12. 验收标准

改造完成后应满足：

- 用户 5 秒内能看到当前月结论，而不是自己算。
- 同一个数字不在多个区域重复抢视觉。
- 年度口径、季度口径、月份口径明确互斥。
- 点击图表只切换当前选择，不误触发项目清单。
- 横坐标月份按钮能说明月份、直营数和加盟数，并打开对应月份清单。
- 0 数据月份不会被误读为经营异常。
- 右侧面板能解释“当前选择是什么、为什么值得看、下一步去哪”。
- 店态和省份排行能承担归因入口，不只是静态图表。
- 2K 桌面端下主图、右侧解释器、底部排行对齐稳定，按钮状态完整。
- 不改变后端统计口径，不改钉钉字段含义和录入校验。

## 13. 推荐决策

推荐直接采用 P0 + P1 的组合方案：

**先把模块重构成经营判断型看板，再补图表和排行联动。**

不建议只做颜色、边框、圆角层面的表面美化。当前问题的根源是信息层级和交互模型不清晰，视觉优化必须和数据取舍、状态统一一起做，才能让这个模块真正从半成品变成可用的首页运营入口。
