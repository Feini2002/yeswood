# 首页年度进店结构大模块改版方案

> 状态：已归档（改版方案草案，已被 V3 取代）  
> 更新日期：2026-06-09  
> 归档日期：2026-06-10  
> 适用范围：首页「年度进店结构」大模块  
> 关联文档：[首页年度进店结构看板需求设计](../17-home-entry-dashboard-design.md)、[V3 落地稿](./2026-06-10-home-entry-structure-v3-plan.md)  
> 核心结论：保留现有统计内容，但把模块从「图表展示」升级为「管理层经营判断 + 运营追踪入口」。

## 1. 模块定位

首页年度进店结构模块不应只是把年度总量、月度柱图、四象限、店态、省份分布摆出来。它应该服务两个真实使用场景：

1. **管理层 30 秒判断**：今年进店规模稳不稳，节奏有没有异常，结构有没有偏，哪里需要解释，下一步找谁。
2. **运营人员 2 次点击定位**：从异常月份、象限、省份或店态进入清单，自动带筛选条件，继续分配跟进和复盘。

因此，本模块的目标不是「看起来像一个图表大模块」，而是形成一条完整路径：

```text
年度总览判断
-> 发现异常月份 / 结构偏移 / 区域集中
-> 点击进入上下文诊断
-> 下钻到项目清单
-> 形成待跟进队列
-> 回到首页观察趋势是否改善
```

## 2. 当前核心问题

### 2.1 没有管理层主叙事

当前模块展示了很多数据，但没有直接回答管理层每天或每周打开首页时最关心的问题：

- 今年进店总盘处在什么水位。
- 最近月是否低于正常节奏。
- 新店 / 老店、直营 / 加盟结构是否健康。
- 哪些月份需要业务解释。
- 哪些省份、店态或象限在拉动或拖累。
- 异常背后应由哪个角色继续处理。

管理层不是来逐项读数的，而是要快速判断风险、责任和下一步动作。

### 2.2 指标卡只有结果，没有经营判断

顶部「总进店、新店、老店、直营、加盟」是必要信息，但如果只是数量卡，用户只能知道「有多少」，不知道「该不该关注」。

这些指标需要补充：

- 占比
- 月均
- 最近月与月均差异
- 峰值月 / 低谷月
- 异常月数量
- 结构集中度
- 可点击清单入口

### 2.3 主图图表语法不清晰

当前主图是「每月两根柱：新店 / 老店；柱内再堆叠直营 / 加盟」。这个表达同时承载时间趋势、店龄结构、直营 / 加盟结构，导致用户必须理解 24 根柱和两套分类关系。

更严重的是，当前主图是 DOM 手写柱图，只有 `max / half / 0` 三段刻度，缺少专业图表的比例尺、网格线、整数刻度、单位、tooltip、选中态、异常标记和联动规则。

### 2.4 四象限没有转成运营语言

本模块的四象限实际是：

```text
              直营            加盟
新店      直营新店        加盟新店
老店      直营老店        加盟老店
```

它不是普通分类，而是四种不同运营打法：

- 直营新店：内部交付节奏、开业节点、跨部门协调。
- 直营老店：翻新效率、停业影响、存量运营恢复。
- 加盟新店：标准复制、加盟商辅导、开业支持。
- 加盟老店：整改配合、标准复核、品牌一致性。

当前用四张卡片平铺，没有矩阵关系，也没有把象限转成「应关注什么」。

### 2.5 排行与分布没有形成行动闭环

店态排行、省份分布不能只是排名。用户看到某个省份或店态异常后，下一步应能进入项目清单，带上筛选条件，再按负责人、阶段、风险继续追。

当前模块更像静态报告，而不是运营入口。

## 3. 管理层核心问题

首页默认态必须优先回答以下问题，按优先级排序：

| 优先级 | 经营问题 | 首页应如何回答 |
| --- | --- | --- |
| P0 | 今年到现在进店总盘有多大 | 年度 / YTD 总进店、月均、最近月 |
| P0 | 最近月是否低于正常水位 | 最近月 vs 月均、最近月 vs 近 3 月均值 |
| P0 | 哪些月份异常 | 主图异常标记、异常月列表、hover 原因 |
| P0 | 结构是否健康 | 新店 / 老店占比、直营 / 加盟占比、四象限占比 |
| P0 | 异常来自哪里 | 主导象限、Top 省份、Top 店态、结构偏移 |
| P1 | 哪些区域或店态需要复盘 | 省份集中度、店态集中度、排名变化 |
| P1 | 下一步找谁 | 按省份、店态、象限映射到责任角色或项目清单 |
| P2 | 是否影响年度目标 | 目标、同比、预测具备后再纳入，不在第一阶段误判 |

第一阶段没有目标值和同比数据时，不做「达标 / 未达标」判断，只做内部经营状态判断：

- 高于自身月均
- 低于近 3 月水位
- 结构较 YTD 出现偏移
- 进店集中度较高
- 该月为疑似异常月，需结合业务动作解释

## 4. 默认态信息优先级

默认态服务「判断」，展开态服务「追责和分析」。

### 4.1 默认态必须一眼看到

- 年度 / YTD 总进店
- 月均进店
- 最近有数据月份及其较月均差异
- 峰值月 / 低谷月
- 新店 / 老店结构
- 直营 / 加盟结构
- 当前月或最近月主导象限
- 异常月数量
- Top 3 省份或店态集中度
- 统一「进店项目清单」入口

### 4.2 点击后再看

- 1-12 月完整明细表
- 单月所有项目明细
- 全量省份列表
- 全量店态列表
- 每个象限下全部项目
- 负责人维度明细
- 数据质量和复杂口径说明
- 未来同比、目标、预测解释

首页不能变成查询表单。默认应让用户知道「今年整体怎么样」，筛选状态再回答「这个时间 / 结构 / 区域为什么这样」。

## 5. 推荐总方案

推荐采用：

**年度经营判断区 + 1-12 月四象限趋势主图 + 上下文诊断侧栏 + 运营归因区 + 项目清单入口**

页面结构如下：

```text
年度进店结构
├─ 顶部：年度经营判断区
│  ├─ 主指标：年度 / YTD 总进店、月均、最近月
│  ├─ 结构指标：新店 / 老店、直营 / 加盟
│  ├─ 节奏指标：峰值月、低谷月、异常月数
│  └─ 动作入口：进店项目清单、待跟进队列（预留）
│
├─ 中部：年度主分析区
│  ├─ 左侧：1-12 月四象限堆叠主图
│  └─ 右侧：上下文诊断侧栏
│
└─ 下部：运营归因区
   ├─ 四象限矩阵
   ├─ 店态 Top 排行
   └─ 省份 Top 分布
```

## 6. 顶部：年度经营判断区

### 6.1 主指标

主指标不只显示年度总进店，还要显示经营水位。

示例：

```text
年度进店 361 项
月均 30 项 · 最近月 15 项 · 低于月均 50%
```

说明：

- `年度进店`：当前选择年份内有效进店项目总量。
- `月均`：优先用 YTD / 已发生月份数；完整年度可用年度总量 / 12。
- `最近月`：最近一个有进店数据的月份，不必等于当前自然月。
- `较月均差异`：最近月与月均的差异，用于第一阶段内部判断。

### 6.2 结构指标

结构指标显示数量和占比：

```text
新店 257 · 71%
老店 104 · 29%

直营 170 · 47%
加盟 191 · 53%
```

建议配两条 100% 结构条：

- 新店 / 老店结构条
- 直营 / 加盟结构条

这些结构条本身可点击，点击后进入结构筛选态，并在顶部生成筛选摘要。

### 6.3 经营判断

顶部右侧放 1-3 条自动生成的短判断。

第一阶段基于内部参照生成，不依赖目标值或同比：

- 最近月是否高于 / 低于月均。
- 峰值月是否过度集中。
- 某象限是否占比超过 50%。
- Top 3 省份或店态是否超过 60%。
- 某月是否较上月波动超过 30%。

示例：

```text
6 月进店低于 YTD 月均 50%，需查看是否为季节性低谷。
直营新店占 6 月进店 67%，本月压力集中在内部交付链路。
Top 3 省份贡献 47%，区域集中度适中。
```

## 7. 1-12 月主图设计

### 7.1 展示判定

推荐：**默认展示 1-12 月全量趋势，但只突出当前月和异常月。**

具体规则：

- 1-12 月一次性展示，不分页。
- 2K 桌面端不横向滚动。
- 所有月份保留年度连续性。
- 当前月或最近有数据月份高亮。
- 异常月加小圆点、警示角标或浅色底纹。
- 其他月份作为背景对比，不抢主视线。
- 季度入口作为辅助模式，不取代默认月份视图。

取舍如下：

| 方案 | 优点 | 缺点 | 结论 |
| --- | --- | --- | --- |
| 12 月全量展示 | 年度节奏完整、对比直接 | 信息密度较高 | 推荐默认 |
| 横向滚动 | 可容纳更多月份 | 隐藏异常，破坏全局感 | 不推荐 |
| 分页 | 空间压力小 | 无法连续比较趋势 | 不推荐 |
| 季度折叠 | 适合经营会看汇总 | 需要二次展开看月 | 作为辅助模式 |
| 关键月聚焦 | 定位效率高 | 丢失年度背景 | 作为分析态 |

### 7.2 主图形态

主图从当前「新店 / 老店双柱 + 直营 / 加盟堆叠」改为：

**每月一根四象限堆叠柱**

每根柱代表一个月总进店，柱内四段：

- 直营新店
- 直营老店
- 加盟新店
- 加盟老店

用户只需要理解一个稳定语法：

- 柱高：当月总进店量。
- 颜色段：四象限结构。
- 横向：月度节奏。
- 高亮：当前选中月份或异常月份。

### 7.3 图表细节

主图应具备：

- 完整 Y 轴整数刻度，单位为「项」。
- 浅色网格线。
- 月份 X 轴。
- 季度分组辅助标识：Q1 / Q2 / Q3 / Q4。
- 月均线。
- 峰值月标记。
- 低谷月标记。
- 当前月 / 最近月标记。
- 异常月标记。
- hover tooltip。
- 点击选中态。

柱顶数值不应每月常驻全部显示。建议只显示：

- 选中月份
- 峰值月
- 异常月
- hover 月份

### 7.4 Tooltip 内容

tooltip 不承载复杂明细，但必须能让用户快速读懂一个月。

示例：

```text
2026 年 6 月 · 总进店 15 项
较 YTD 月均 -50% · 环比 -32%

新店 10 项，占 67%
老店 5 项，占 33%
直营 11 项，占 73%
加盟 4 项，占 27%

直营新店 10
直营老店 1
加盟新店 3
加盟老店 1

店态 Top：常规店 7、下沉店 5
省份 Top：广东省 3、安徽省 2、江苏省 2
```

### 7.5 图表库选择

本次改版明确抛弃当前年度进店结构主图的自研 DOM 柱图，改用成熟图表库承载主图交互和比例表达。

当前已安装：

- `echarts`：`6.1.0`
- 安装位置：`package.json` 的 `devDependencies`
- 锁文件：`pnpm-lock.yaml`
- 传递依赖：`zrender 6.1.0`、`tslib 2.3.0`

注意：仓库现有策略要求第三方前端库不能进入 `dependencies`，因此 ECharts 放在 `devDependencies`。后续实现时不能直接假设浏览器能访问 `node_modules`，需要在第一阶段确定前端运行时加载方式。

ECharts 主图配置方向：

- `series`: 4 个 bar series，统一 `stack: 'entry'`
- `xAxis`: 1-12 月 category
- `yAxis`: value，`minInterval: 1`
- `tooltip`: axis trigger，shadow pointer
- `legend`: 直营新店 / 直营老店 / 加盟新店 / 加盟老店
- `markLine`: 月均线
- `markPoint`: 峰值月
- `markArea`: 仅用于季度或异常背景，不做装饰性使用
- 不建议默认启用 `dataZoom` 和 `brush`

前端加载策略建议二选一：

1. 引入轻量构建步骤，把 ECharts 按需打包到可服务的前端模块。
2. 增加明确的 vendor 产物策略，并同步调整现有「不允许 `public/vendor`」的策略测试。

不建议临时从 `node_modules` 直接暴露静态路由。这样虽然开工快，但会绕过当前静态服务边界和前端拆分策略。

## 8. 时间维度交互

### 8.1 时间层级

时间状态分三层，UI 上必须区分：

1. **年份上下文**：模块最高层时间范围，默认当前年份。
2. **时间范围**：全年、季度或月份。
3. **单月焦点**：当前选中或 hover 的月份。

不要让用户误以为 hover 某月就是全局筛选。hover 只是读数，点击才改变分析上下文。

### 8.2 年份

- 年份选择放在模块标题区右侧。
- 切换年份后，年度总览、月度趋势、四象限、店态排行、省份分布全部刷新。
- 切换失败时保留上一份可读数据，并在标题区显示错误提示。

### 8.3 季度

季度是辅助组织方式，不作为重筛选器常驻铺开。

建议在主图上方或 X 轴附近提供轻量控件：

```text
全年 | Q1 | Q2 | Q3 | Q4
```

点击季度后：

- 主图保留 1-12 月背景。
- 选中季度内月份高亮。
- 右侧诊断切换为季度诊断。
- 四象限、店态、省份按该季度聚合。
- 顶部筛选摘要显示 `2026 年 · Q2`。

### 8.4 月份

点击某月后：

- 主图保留全年背景。
- 被选月份高亮。
- 右侧诊断切换为该月。
- 四象限矩阵、店态排行、省份分布全部切换到该月口径。
- 顶部筛选摘要显示 `2026 年 · 6 月`。

再次点击已选月份，可以取消该月份筛选，恢复全年或当前上层时间范围。

### 8.5 异常月

异常月不应藏在筛选器里，应直接标在主图上。

hover 异常标记显示：

- 异常类型
- 异常程度
- 可能来源

点击异常标记后进入异常诊断：

- 是总量异常还是结构异常。
- 是否由某象限、某省份、某店态拉动。
- 是否需要进入项目清单。

## 9. 上下文诊断侧栏

右侧不只是「选中月份数字面板」，而是上下文诊断侧栏。

### 9.1 默认总览态

默认展示全年 / YTD 诊断：

- 年度总进店
- 月均
- 最近月与月均差异
- 峰值月 / 低谷月
- 异常月数量
- 主导象限
- Top 省份 / 店态集中度

### 9.2 时间聚焦态

选择季度或月份后，侧栏展示该时间范围：

- 总进店
- 较月均或近 3 月均值差异
- 环比
- 新店 / 老店占比
- 直营 / 加盟占比
- 主导象限
- Top 省份
- Top 店态
- 建议动作入口

### 9.3 结构筛选态

点击象限、店态或省份后，侧栏展示该对象在当前时间范围下的贡献：

- 数量
- 占比
- 与 YTD 结构对比
- 是否集中
- 项目清单入口

### 9.4 明细入口

侧栏不直接铺项目表。它只提供：

- `查看项目清单`
- `加入待跟进队列`（第二阶段）
- `按负责人分组`（预留）
- `导出会议材料`（预留）

项目表进入抽屉或清单页，避免首页被无限撑长。

## 10. 运营归因区

### 10.1 四象限矩阵

四象限改成 2 x 2 经营矩阵。

```text
              直营            加盟
新店      直营新店        加盟新店
老店      直营老店        加盟老店
```

每格显示：

- 数量
- 占当前时间范围比例
- Top 店态
- Top 省份
- 运营含义短句

示例：

```text
直营新店 10 项 · 占 67%
主力店态：常规店
主力省份：广东省
关注：内部交付节奏与开业节点
```

点击象限后进入结构筛选态。

### 10.2 店态排行

首页只展示 Top 8 + 其他。

每行显示：

- 店态名
- 数量
- 占比
- 结构拆分提示

点击店态行后：

- 顶部筛选摘要增加 `店态：X`
- 侧栏切换为店态诊断
- 可进入项目清单

### 10.3 省份分布

首页展示 Top 10 + 其他。

每行显示：

- 省份名
- 数量
- 占比
- 新店 / 老店或直营 / 加盟结构

同时显示 Top 3 集中度：

```text
Top 3 省份贡献 47%
```

点击省份后进入区域/排行下钻态。

## 11. 筛选和状态规则

### 11.1 常驻控件

首页只常驻三类控件：

- 年份
- 时间范围：全年 / Q1 / Q2 / Q3 / Q4
- 核心结构维度：四象限或店态

省份、城市、负责人、更多店态等维度进入侧栏或高级筛选，不在首页顶部铺开。

### 11.2 筛选摘要

一旦用户点击月份、季度、象限、店态或省份，模块顶部生成筛选摘要：

```text
2026 年 · Q2 · 6 月 · 直营新店 · 广东省
```

规则：

- 每个条件可单独移除。
- 提供「恢复全年总览」入口。
- 清除后回到默认态，但当前月和异常月标记仍保留。

### 11.3 Hover / Click / Legend

- hover 只解释，不改变数据状态。
- click 才进入选中态并联动其他模块。
- 再次 click 已选对象，取消选中。
- legend 点击只影响图表可见性，不改变总览统计口径。
- 象限、店态、省份点击属于业务筛选，会改变侧栏和归因区口径。

### 11.4 联动优先级

多条件并存时，筛选优先级为：

```text
年份 > 时间范围 > 象限 > 店态 > 省份
```

被筛选命中的对象高亮，未命中但仍有参考价值的数据置灰，不要直接消失。只有用户选择「仅看筛选结果」时，才隐藏背景数据。

## 12. 具体数据查看方式

### 12.1 Tooltip

用于快速读数：

- 单月数值
- 占比
- 环比
- 较月均差异
- Top 省份 / 店态
- 异常原因摘要

tooltip 不承载复杂明细。

### 12.2 侧栏

用于首页内持续分析：

- 点击月份
- 点击季度
- 点击异常标记
- 点击象限
- 点击省份
- 点击店态

侧栏展示摘要、结构拆解、Top 变化、相关异常和下钻入口。

### 12.3 抽屉

用于查看较多明细，但不离开首页：

- 项目列表
- 城市拆分
- 店态明细
- 负责人分组

抽屉支持排序、搜索、导出。关闭抽屉后回到原分析上下文，不丢筛选条件。

### 12.4 跳转

用于进入完整分析页或专项页面。跳转是二级动作，不是查看具体数据的第一选择。

推荐组合：

```text
hover tooltip 解决快速读数
-> 点击侧栏解决上下文解释
-> 侧栏内打开抽屉解决明细表
-> 抽屉内再跳转解决专项深挖
```

## 13. 前端展示与控件收束

本节用于约束第一阶段真实实现时的前端展示逻辑。核心原则是：默认观看要顺，操作入口要轻，复杂信息进入侧栏或抽屉，不用弹窗打断用户判断。

### 13.1 总体观看路径

用户视线应按固定顺序移动：

```text
顶部经营判断
-> 1-12 月主图
-> 右侧上下文诊断
-> 下方四象限 / 店态 / 省份归因
-> 清单抽屉或专项跳转
```

实现时不要把控件放得比数据更抢眼。用户第一次打开页面时，应该先看到「今年怎么样」，不是先看到一排筛选器。

### 13.2 下拉框与分段控件

不同控件承担不同职责：

| 控件 | 用途 | 展示规则 |
| --- | --- | --- |
| 年份下拉框 | 切换年度上下文 | 放标题右侧，单选，默认当前年 |
| 时间分段控件 | 全年 / Q1 / Q2 / Q3 / Q4 | 放主图上方或 X 轴附近，不做复杂多选 |
| 月份选择 | 通过主图点击完成 | 不额外做 1-12 月下拉框 |
| 结构维度切换 | 四象限 / 新老店 / 直营加盟（可选） | 用分段控件，不用下拉框 |
| 更多筛选 | 省份、店态、负责人等高级条件 | 用轻量 popover 或侧栏，不常驻铺开 |

月份不建议做下拉框。月度是图表自身的交互对象，用户应通过点击柱体或异常标记进入月份分析。额外下拉会制造两个入口，增加状态理解成本。

季度也不建议做成下拉框。季度是快速视角切换，用分段控件更容易扫描，也更符合 2K 桌面看板使用。

### 13.3 Tooltip 细节

tooltip 只解决「看一眼」：

- hover 延迟不应过长，建议 150-250ms 内出现。
- 不遮挡当前柱体和当前行，优先向右或向上展开。
- 内容最多分三组：总量、结构、Top 来源。
- 不放按钮，不承载复杂操作。
- 移动鼠标离开后自动关闭。
- 数字、占比和异常原因要比说明文字更醒目。

tooltip 不替代侧栏。用户如果需要继续看原因、筛选或清单，应点击进入侧栏。

### 13.4 侧栏细节

右侧侧栏是本模块最重要的上下文解释区，不是装饰卡片。

侧栏状态：

- 默认总览态：展示全年 / YTD 经营判断。
- 时间聚焦态：展示选中季度或月份。
- 异常诊断态：展示异常类型和来源拆解。
- 结构筛选态：展示选中象限、店态或省份。

侧栏展示规则：

- 宽度稳定，不因内容长短抖动。
- 标题区固定显示当前分析对象，例如 `2026 年 6 月`、`Q2`、`广东省`。
- 标题下方显示筛选摘要。
- 中部放 5-7 个关键判断，避免长篇说明。
- 底部放动作入口：`查看项目清单`、`按负责人分组`、`加入待跟进`（预留）。
- 侧栏内不要直接放长表格。

用户点击不同对象时，侧栏内容切换，但主图的全年背景不消失。这样用户知道自己仍在年度上下文里。

### 13.5 抽屉细节

抽屉用于「看具体数据」，比弹窗更适合本模块。

触发入口：

- 侧栏中的 `查看项目清单`
- 顶部统一 `进店项目清单`
- 店态排行行
- 省份排行行
- 四象限格子

抽屉建议：

- 从右侧打开，保留首页背景。
- 宽度适合表格阅读，2K 桌面建议 720-960px。
- 顶部固定来源和筛选条件。
- 支持关闭后回到原分析上下文，不清空筛选。
- 不建议点击遮罩误关闭，避免用户在清单中操作时丢上下文。
- 提供明确关闭按钮和 `返回看板` 操作。

抽屉头部示例：

```text
项目清单
来源：首页 > 年度进店结构 > 2026 年 6 月 > 广东省
筛选：年份 2026 | 月份 6 月 | 省份 广东省 | 进店项目 有效
```

抽屉表格首屏字段：

- 项目名称
- 省份 / 城市
- 店态
- 新店 / 老店
- 直营 / 加盟
- 进店月份
- 当前阶段
- 负责人
- 最近更新时间
- 操作入口

抽屉内可提供二级筛选，但不要复制首页全部筛选器。优先提供：

- 搜索项目 / 门店
- 只看异常
- 按负责人分组
- 导出

### 13.6 弹窗使用边界

本模块不建议用弹窗承载核心分析。

弹窗只用于：

- 口径说明
- 删除 / 清除筛选确认
- 加入待跟进队列的轻量确认
- 单条记录的短说明

弹窗不用于：

- 月份分析
- 省份分析
- 店态分析
- 项目清单
- 四象限明细

原因是弹窗会遮住主图和上下文，用户无法边看趋势边判断原因。

### 13.7 加载、空、错误与禁用状态

专业看板必须把边界状态设计清楚。

加载态：

- 年份切换时保留旧数据，侧栏或标题区显示刷新中。
- 不要整块白屏。
- 主图可降低透明度，避免用户误以为数据已切换完成。

空状态：

- 某年无进店数据：显示年度空状态和年份选择。
- 某月无数据：主图该月为 0 基线，不显示最小假柱。
- 省份或店态为空：展示「暂无可归因数据」，不要隐藏整个区域。

错误态：

- 年份接口失败时保留上一年数据。
- 标题区显示错误提示和重试入口。
- 不改变当前筛选摘要。

禁用态：

- 无数据对象不可点击。
- 被 legend 隐藏的系列不进入清单筛选。
- 高级筛选条件无命中时，清单入口显示 0 项并可清除条件。

### 13.8 文案和操作反馈

操作反馈要明确、短、可理解：

- 点击月份后，筛选摘要立即更新。
- 点击象限后，侧栏标题明确写出象限含义。
- 清除某个筛选条件后，相关图表恢复对应口径。
- `查看项目清单` 进入抽屉前，按钮旁可显示数量，例如 `查看项目清单 15`。
- 不使用「查看更多」这种模糊文案，改为「查看本月项目」「查看广东省项目」「查看加盟老店项目」。

## 16. 派生指标与诊断规则

### 16.1 指标优先级

| 优先级 | 指标 | 口径 | 管理价值 |
| --- | --- | --- | --- |
| P0 | 年度 / YTD 进店量 | 当前年份有效进店项目累计 | 判断总盘规模 |
| P0 | 月均进店量 | YTD / 已发生月份数 | 判断正常水位 |
| P0 | 最近月进店量 | 最近一个有进店数据的月份 | 判断当前状态 |
| P0 | 最近月 vs 月均 | 最近月 / 月均 - 1 | 判断高低水位 |
| P0 | 峰值月 / 低谷月 | 月度最大 / 最小值 | 判断节奏依赖 |
| P0 | 环比 | 本月 / 上月 - 1 | 判断短期变化 |
| P0 | 季度累计 | Q1-Q4 各自月份累计 | 支持经营会视角 |
| P0 | 结构占比 | 四象限、店态、省份各自占比 | 判断结构来源 |
| P1 | 近 3 月均值 | 最近 3 个完整月份平均 | 降低单月误判 |
| P1 | Top 3 集中度 | Top 3 省份 / 店态 / 象限占比 | 判断依赖集中 |
| P1 | 结构偏移 | 最近月占比 - YTD 占比 | 判断结构变化 |
| P1 | 异常月识别 | 偏离月均、环比或结构阈值 | 快速定位解释对象 |
| P2 | 波动系数 | 月度标准差 / 月均 | 判断节奏稳定性 |
| P2 | Run-rate 预测 | 近 3 月均值 x 12 | 目标接入前的估算参考 |

### 16.2 展示层级

| 展示层级 | 适合内容 |
| --- | --- |
| 常驻显示 | 年度 / YTD 总量、月均、最近月、峰值月、季度累计、结构占比、异常月数 |
| Tooltip | 单月数量、占比、环比、较均值偏差、Top 省份 / 店态 |
| 诊断侧栏 | 异常月列表、结构偏移解释、集中度提示、Top 变化 |
| 抽屉 / 清单 | 项目明细、负责人、阶段、风险、下一动作 |

### 16.3 异常规则

第一阶段规则保持简单、可解释：

```text
总量异常：
当月进店量较 YTD 月均偏离超过 25%，标记为总量异常。

环比异常：
当月进店量较上月变化超过 30%，标记为环比异常。

结构异常：
某分类当月占比较 YTD 占比变化超过 10 个百分点，标记为结构偏移。

集中度提示：
Top 3 省份 / 店态 / 象限贡献占比超过 60%，标记为集中度较高。
```

文案必须克制，避免没有目标或同比支撑时做过度判断：

- 使用「高于自身月均」「低于近 3 月水位」「疑似异常」「需进一步解释」。
- 避免「达标」「未达标」「增长好」「增长差」。

## 17. 视觉规范建议

### 17.1 整体气质

这是空间视觉项目运营中台，不适合营销式大卡片或装饰性背景。

视觉方向：

- 克制
- 高密度
- 专业
- 可扫描
- 数据可信

### 17.2 配色

建议用两套色相加深浅表达四象限：

- 直营：绿色系
- 加盟：蓝色系
- 新店：深色
- 老店：浅色

示例：

- 直营新店：深绿
- 直营老店：浅绿
- 加盟新店：深蓝
- 加盟老店：浅蓝

避免过重渐变。B 端图表应以稳定实色、浅网格、清楚标签为主。

### 17.3 信息密度

2K 桌面端可以提高信息密度，但扫描路径必须清楚：

- 主图高度建议 300-360px。
- 顶部判断区不超过一行半。
- 右侧侧栏信息控制在 5-7 个关键点。
- 下方归因区不做大面积留白。
- 省份和店态只展示 Top。
- 长文本进 tooltip 或侧栏，不堆在卡片中。

## 18. 工程落地建议

后端统计口径第一阶段不改，继续使用现有 payload：

- `payload.year`
- `payload.defaultMonth`
- `payload.totals`
- `payload.months`
- `months[].newStore`
- `months[].oldStore`
- `months[].quadrants`
- `months[].storeStatuses`
- `months[].provinces`

前端需要新增 view model 层，把展示判断从字符串模板里抽出来。

建议拆分：

- `entryStructureViewModel(payload)`：整理 series、月均、峰值、低谷、季度、异常、Top 数据。
- `renderEntrySummaryKpis(viewModel)`：顶部经营判断区。
- `renderEntryStructureChart(viewModel, state)`：主图。
- `renderEntryContextInspector(viewModel, state)`：右侧诊断侧栏。
- `renderEntryQuadrantMatrix(viewModel, state)`：四象限矩阵。
- `renderEntryRankingPanels(viewModel, state)`：店态与省份排行。
- `buildEntryDrillFilter(state, source)`：生成项目清单筛选条件。

交互状态建议：

```text
state = {
  selectedYear,
  selectedTimeRange, // all | quarter | month
  selectedQuarter,
  selectedMonth,
  selectedQuadrant,
  selectedStoreStatus,
  selectedProvince,
  visibleSeries,
  inspectorMode, // overview | time | anomaly | quadrant | province | status
}
```

主文件可先继续放在 `public/dashboard/annual-entry-structure.mjs`，但不要继续让一个函数同时承担状态、数据计算、图表绘制、详情布局和交互绑定。

## 19. 分阶段实施

### 第一阶段：重做为可用经营看板

目标：不改后端，先让首页模块真实可用。

范围：

- 新增 view model。
- 确定 ECharts 前端加载策略，避免浏览器无法访问 `node_modules`。
- 顶部改为经营判断区。
- 主图改为 1-12 月四象限堆叠柱。
- 明确当前月、峰值月、低谷月、异常月。
- 增加季度轻量切换。
- 增加右侧上下文诊断侧栏。
- 补齐 tooltip、侧栏、抽屉、弹窗、下拉框和空/错/加载状态规则。
- 四象限改为 2 x 2 矩阵。
- 店态、省份改为 Top 归因区。
- 数字、月份、象限、排行行可点击。
- 生成项目清单筛选条件。
- 不恢复大块「数据说明」栏目。

### 第二阶段：补运营信号和待跟进入口

范围：

- 环比
- 近 3 月均值
- 结构偏移
- 区域集中度
- 店态集中度
- 异常月列表
- 待跟进队列入口
- 负责人 / 风险 / 验收入口预留

### 第三阶段：接入目标、同比和预测

范围：

- 年度目标
- YTD 目标达成率
- 同比
- 去年同期参考线
- Run-rate 预测
- 年度缺口
- 会议导出材料

这部分需要确认目标来源、同比数据口径和责任边界，不建议第一阶段强行做。

## 20. 验收标准

改版后至少满足：

- 管理层 30 秒内能判断年度规模、最近水位、结构主因和异常入口。
- 1-12 月默认全量可见，不分页、不横滚，但当前月和异常月突出。
- 点击季度、月份、异常标记后，侧栏和归因区同步切换。
- 点击象限、店态、省份后，顶部出现筛选摘要，并能恢复全年总览。
- tooltip 用于快速读数，侧栏用于上下文解释，抽屉 / 清单用于具体明细。
- 年份用下拉框，季度用分段控件，月份通过主图点击，不额外做月份下拉。
- 弹窗不承载核心分析，项目明细进入抽屉或清单页。
- 所有关键数字都能进入项目清单或至少生成明确筛选条件。
- 首页不堆满筛选器，仍保持经营看板而不是查询表单。
- 空数据月不会显示误导性最小柱。
- 2K 桌面端布局紧凑、稳定、无文字重叠。
- 不改变后端统计口径，不破坏现有年度进店结构测试。

## 21. 推荐决策

推荐直接采用第一阶段方案：

**1-12 月四象限堆叠主图 + 管理层经营判断区 + 上下文诊断侧栏 + 可点击归因区 + 项目清单筛选继承**

这个方案比单纯美化图表更重要，因为它解决的是未来真实使用问题：

- 默认能看全局。
- 异常能被发现。
- 点击能解释原因。
- 清单能继承筛选。
- 后续能接待跟进、负责人、风险和验收。

不推荐继续沿用当前双柱 DOM 图。它的问题不是样式细节，而是图表语法、交互状态和运营闭环都不足。
