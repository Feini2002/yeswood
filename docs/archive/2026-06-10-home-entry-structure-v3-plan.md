## 1. 背景与目标

V1/V2 讨论已达成共识：当前模块存在为「填满版面」而叠放的重复解读层（节奏卡、侧栏解释器、占比、月份下拉等），与真实使用场景不符。

V3 在 v2「减法共识」之上，补齐三块落地规格：

1. **季度 tab 带项数**，且 UI 对齐系统整体调性（非线框草稿样式）。
2. **主图 ECharts 丰满化**，月份轴 `直/盟` 改为 `直营/加盟`，全模块字号上调 1～2px。
3. **其余 v2 建议全部纳入**（单行状态带、弹窗精简、空月静默、点击统一等）。

核心原则：

| 原则 | 说明 |
| --- | --- |
| 跟系统走，不另起炉灶 | 季度 tab 对齐 `details-view-tabs` / `filter-toggle-button` 的绿白切换；年份选择继续用现有 `native-filter-select` 外观 |
| 数字为主，不加解读层 | 一行状态带 + tab 带项数，不再要侧栏和节奏卡 |
| 图表承担节奏，文字承担结构 | 主图讲「什么时候、多少」；顶部一行讲「新店/老店/直营/加盟」 |
| 全模块 +1～2px | 11→12、12→13、13→14 分档上调，2K 桌面可读性优先 |

## 2. 信息架构变更

### 2.1 删除

- 右上角**月份下拉**（只保留年份选择）。
- 决策带三卡：当前聚焦卡、节奏判断卡、带 `%` 的结构条。
- 右侧解释器整栏（`renderContextInspector`）。
- 弹窗顶部 4 张摘要卡（当前清单 / 直营项目 / 新店项目 / 月份范围）。
- 「当前查看：xxx」重复行。
- 所有 **%** 与自动点评文案（如「新店驱动明显」「直营/加盟基本均衡」）。

### 2.2 保留并强化

- 年份选择（右上唯一全局筛选）。
- **单行状态带**（随 scope 联动变化）。
- **季度 segmented control**（带项数，新 UI）。
- **主图全宽**（ECharts 视觉加强）。
- 店态分布 + 省份贡献（标题精简，随 scope 联动）。
- **主图月份按钮 + 项目弹窗**（四象限筛选保留）。

## 4. 单行状态带

一行读完当前 scope，不拆两行：

```text
2026年全年 · 374项  │  新店 267  老店 107  直营 177  加盟 197
```

规格：

- **左段**（scope + 总量）：`14px / font-weight 950`；总量数字可用 `17px` 略强调。
- **竖线** `│` 分隔左右。
- **右段**四个结构数：`13px / 900`；标签灰色、数字墨绿 `#104020`。
- 底部分割线 `1px var(--ov-border)`，与 tab 区轻分区；不加卡片边框。

联动示例：

| scope | 左段文案 |
| --- | --- |
| 全年 | `2026年全年 · 374项` |
| Q1 | `2026年 Q1 · 198项` |
| 3 月 | `2026年 3月 · 94项` |

右段四数随 `buildContext` 当前口径重算。

## 5. 季度 Tab UI（Segmented Scope Switch）

### 5.1 问题

现有 5 个独立小方钮 + `Q1` / `1-3月` 分两列展示，视觉像临时拼接，与首页 `details-view-tabs`、筛选按钮的绿白激活态不一致。

### 5.2 设计方案

**容器** `entry-structure-scope-switch`：

- 浅绿底 `#F4FAF5`
- 边框 `1px var(--ov-border)`
- 圆角 `8px`，内边距 `3px`
- 5 段等宽，`gap: 2px`

**单个 segment（两行信息）**：

```text
  全年          Q1           Q2
  374项        198项        176项
  1-12月       1-3月        4-6月
```

| 状态 | 样式 |
| --- | --- |
| 默认 | 透明底；标题 `13px/900`；项数 `15px/950` + `tabular-nums`；副标题 `11px` 弱化 |
| 悬停 | 白底 + 浅绿描边（同 `filter-toggle-button:hover`） |
| 选中 | 白底 + `var(--brand-green)` 描边 + 轻阴影；标题品牌绿；项数 `#104020` |
| 无数据 | 项数显示 `—`；整体 `opacity: 0.55`；仍可点击 |

**不采用**讨论稿中的 `[全年 374]` 方括号线框——那是示意，落地用系统 segmented 语言。

**系统参照**：

- 激活色逻辑：`details-view-tabs button.active`
- 容器质感：`filter-toggle-button` 浅绿 hover 区
- 数字字重：`overview-command-facts dd`（`tabular-nums`）

副标题映射：

| key | 标题 | 副标题 |
| --- | --- | --- |
| all | 全年 | 1-12月 |
| q1 | Q1 | 1-3月 |
| q2 | Q2 | 4-6月 |
| q3 | Q3 | 7-9月 |
| q4 | Q4 | 10-12月 |

项数取自各季度 `createAggregateContext` 的 `total`；全年为 `payload.totals.entry`。

## 6. 主图 ECharts 丰满化

在现有 `buildMainChartOption` 上加强，不更换图表库。

### 6.1 柱体

| 参数 | 现状 | V3 |
| --- | --- | --- |
| `barWidth` | 16 | **22** |
| `barCategoryGap` | 28% | 略收紧，柱组更饱满 |
| `borderRadius` | 四象限圆角 | 保持；顶柱 `[4,4,0,0]` |
| 柱顶标签 | 无 | `total > 0` 时显示 `{c}项`，`12px/900` |
| `emphasis` | `focus: series` | 增加 `scale: 1.02` |

### 6.2 月份按钮（x 轴 rich label）

**文案**：`直` / `盟` → **`直营` / `加盟`**

有数据月份（两行）：

```text
  3月
直营 28  加盟 45
```

无数据月份（7–12 月）：

```text
  7月
    —
```

不显示 `直营 0 加盟 0`。

尺寸调整：

| 属性 | 现状 | V3 |
| --- | --- | --- |
| 月份按钮宽 | 48px | **58px** |
| 月份按钮高 | 22px | **26px** |
| 渠道行字号 | 11px | **12px** |
| 选中态 | 浅绿底 + 绿描边 | 保留 |

### 6.3 图例与坐标

- 图例：`12px → 13px`；色块 `11→12px`；`itemGap` 加大。
- Y 轴：`11px → 12px`；`splitLine` 保持 `#EEF1EC`。
- Tooltip：维持四象限明细，字号 `+1px`。
- `grid.bottom`：`66 → 78`（容纳加高的月份区）。
- `chartTextStyle` 默认：`11 → 13`。

### 6.4 选中态

- 选中月 / 当前季度：`markArea` 浅绿竖带（`rgba(31, 122, 69, 0.08)`，保留）。
- 选中月柱：`borderColor #1F2A22`（保留）。

## 8. 交互规则

| 操作 | 行为 |
| --- | --- |
| 点击**月份按钮**（x 轴） | 只打开该月项目弹窗；外部主图与底部图保持当前 scope |
| 点击**柱子** | 与月份按钮一致：只打开弹窗，不切外部 scope |
| 点击**季度 tab** | 切季度 scope，**关闭弹窗** |
| 点击**全年** | 回到全年 scope，关闭弹窗 |
| 拖动**数据滚动条** | 切为滚动条可视月份范围，店态分布与省份贡献按该范围重新聚合，关闭弹窗 |
| 切换**年份** | 回到全年，关闭弹窗，重新拉数 |

状态机保持现有字段，删除月份下拉后不再通过 `contextMode: 'month'` 驱动外部单月 scope；主图点击只写入 `modal.month` 并打开弹窗。

```text
selectedYear
contextMode: year | quarter | range
selectedQuarter: all | q1 | q2 | q3 | q4
selectedMonth: 0
chartViewport: { startMonth, endMonth }
modal: { open, filter, storeStatus, month }
```

## 10. 底部分布区

保留店态 + 省份两张图，店态从排行改为当前口径分布：

| 现状 | V3 |
| --- | --- |
| `店态排行 · 2026年全年` | **`店态分布`** |
| `省份排行 · 2026年全年` | **`省份贡献`** |

店态分布展示当前 scope（全年 / 季度 / 数据滚动条范围）内所有有数据店态，不做 Top 合并；点击月份时只让弹窗按单月筛选，底部图仍保持外部 scope。数据随当前 scope 联动，不在标题重复 scope 文案。

点击店态条打开项目详情弹窗，并按被点击店态筛选项目列表；弹窗内保留四象限筛选作为二级细分。

## 13. V3 相对 V2 的增量摘要

1. 季度 tab 有完整 UI 规格，不是仅「带个数字」。
