# 源氏木语品牌 UI 参考（轻量版）

## 核心判断

这个 SaaS 系统服务的是空间视觉部门的线下门店项目管理，不是官网、品牌宣传页、门店海报系统。品牌参考只需要做到“看得出是源氏木语内部系统”，不需要把品牌口号、生活方式叙事、海报语言硬塞进每个页面。

更准确的方向是：项目管理工具的骨架 + 源氏木语的颜色与气质。系统要先好用、清楚、稳定，再通过 logo、深绿主色、暖白背景、木色辅助，让它自然贴近品牌。

## 设计边界

| 可以借用 | 不建议借用 |
| --- | --- |
| 官网深绿色品牌识别。 | 大面积官网式营销首屏。 |
| logo 放在侧边栏或登录页。 | 每个页面反复出现 slogan。 |
| 温和、自然、可信的颜色气质。 | 家居海报式排版。 |
| 暖白、木色、浅绿作为背景和辅助色。 | 产品图、样板间图作为数据看板背景。 |
| 简洁中文信息层级。 | 为了“品牌感”牺牲表格和状态可读性。 |

## 推荐色板

| 角色 | 色值 | 用法 |
| --- | --- | --- |
| 品牌主绿 | `#104020` | 侧边栏、主按钮、logo 承托底色。 |
| 品牌柔和绿 | `#2A543B` | 顶部栏、hover、较大面积深色背景。 |
| 操作绿 | `#317754` | 链接、选中态、成功状态、主图表。 |
| 浅绿底 | `#DCEBE2` | 筛选选中态、轻提示、表格 hover。 |
| 页面暖白 | `#F7F3EC` | 工作区背景，替代冷灰背景。 |
| 卡片白 | `#FFFFFF` | KPI、图表、表格容器。 |
| 木色线条 | `#DDD4C8` | 边框、分割线、输入框边界。 |
| 正文深灰 | `#333333` | 标题和正文主文字。 |
| 辅助灰 | `#6F6A61` | 次要说明、更新时间、空状态。 |
| 风险红 | `#C2412D` | 延期、高风险标签，控制用量。 |
| 预警黄 | `#B7791F` | 临近风险、待确认状态。 |

使用比例建议：白/暖白为主，深绿作为品牌锚点，木色只做辅助，风险色只服务状态表达。

## Logo 使用

- 使用 `yeswood-logo.png` 时必须放在深绿底上，因为它是透明底白色字标。
- 侧边栏顶部可以放 logo，下方用小字写“空间视觉项目中台”或“项目状态看板”。
- 如果空间较小，用 `yeswood-favicon.jpg` 作为方形标识即可。
- 不要把 logo 放到图表中、表格中、每张卡片里；系统不是品牌展示板。

## 字体和排版

- 字体优先：`Source Han Sans SC`、`Noto Sans CJK SC`、`Microsoft YaHei UI`、`PingFang SC`。
- 页面标题控制在 28-32px，模块标题 16-18px，表格正文 13-14px。
- 信息密度可以比官网高，因为这是工作系统；留白服务阅读，不服务“高级感表演”。
- 英文可以少量保留，如 `Readonly`、`Dashboard`，但中文应是主信息。

## 对当前系统的落地方式

| 当前区域 | 建议 |
| --- | --- |
| 左侧导航 | 改成品牌深绿，顶部放 logo，导航项保持简洁。 |
| 顶部标题 | 用“空间视觉项目看板”这类业务标题，不写宣传口号。 |
| 筛选区 | 保持工具属性，白底、细边框、品牌绿 focus。 |
| KPI 卡片 | 用顶部细色条区分类型，不做复杂装饰。 |
| 图表 | 主色用绿，辅助用木色，风险用红/黄。 |
| 明细表 | 可读性优先，hover 用极浅绿或暖白，不要大面积彩底。 |
| 空状态 | 用一句业务文案即可，例如“暂无符合条件的项目”。 |

## 一句话原则

让系统“像源氏木语的人在用”，而不是“像把源氏木语官网搬进后台”。
