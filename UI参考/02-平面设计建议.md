# 平面视觉边界与轻量建议

## 先明确边界

这个 SaaS 系统不需要承担太多平面设计任务。空间视觉部门虽然做线下门店，但系统本身的核心是项目状态、节点、风险、资料和协同，不是门店海报、活动物料或品牌宣传。

所以平面视觉在这里最多只做三件事：

1. 提供基本品牌识别：logo、主色、字体气质。
2. 让数据页面不冷、不散、不像通用模板。
3. 给少量导出物或汇报页提供统一封面/标题样式。

## 系统内不必做的事

- 不必每个页面都出现 slogan。
- 不必把门店物料、海报、样板间视觉搬进系统界面。
- 不必使用大面积产品图做背景。
- 不必为每个模块做品牌图形装饰。
- 不必为了“有设计感”降低表格、筛选、状态标签的清晰度。

## 可保留的轻量平面元素

| 元素 | 建议 |
| --- | --- |
| 登录页 | 可以使用深绿底 + logo + 简短系统名。 |
| 侧边栏 | logo、品牌绿、简洁导航即可。 |
| 页面标题区 | 用清楚的业务标题，不做海报式大标题。 |
| 导出 PDF/PPT 封面 | 统一 logo、项目名、门店名、日期、负责人。 |
| 空状态 | 可用极简线性图标，不需要插画。 |
| 标签体系 | 用统一颜色表达项目阶段、风险、资料状态。 |

## 颜色使用方式

- 品牌绿用于框架：侧边栏、主按钮、选中态。
- 暖白用于工作区：让系统比冷灰后台更贴近家居品牌。
- 木色用于辅助：边框、分隔、低优先级标签。
- 红黄只用于风险：不要参与品牌氛围。

## 导出物建议

如果后续系统支持导出项目汇报或节点报告，可以保持简单模板：

- 封面：logo + 项目名称 + 门店/城市 + 日期。
- 内页：项目基础信息、节点进度、风险列表、待确认事项。
- 图表：继续沿用系统色板，避免每次导出重新设计。
- 页脚：项目编号、数据更新时间、只读来源说明。

## 给设计和前端的共识

平面设计在这个系统里是“秩序感”和“品牌温度”的辅助，不是主角。真正的主角是线下门店项目是否看得清、查得到、追得动、复盘得了。
