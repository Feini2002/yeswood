# 前端界面与组件小建议

## 前端方向

结合前端设计 skill 的思路，这个系统不适合做成营销页或装饰型界面。它更适合走“克制的桌面端内部运营工具”方向：结构稳定、信息密度合理、按钮明确、状态颜色可靠，同时用源氏木语的品牌绿和暖白木色建立轻量识别。

关键词：清楚、耐看、可扫描、轻品牌化、桌面端优先。

## 后端字段口径

当前后端已经把钉钉数据整理成只读项目数据，前端布局应围绕这些真实字段设计：

| 字段 | 含义 | 适合出现位置 |
| --- | --- | --- |
| `name` | 项目/门店名称 | 明细表、风险列表、详情页标题。 |
| `province` | 省份 | 筛选、区域分布、明细表。 |
| `businessType` | 业态 | 筛选、业态分析、明细表。 |
| `storeStatus` | 店态 | 筛选、店态分析、明细表。 |
| `status` | 项目状态 | 筛选、状态分布、状态标签。 |
| `owner` | 负责人 | 筛选、负责人负载、明细表。 |
| `progress` | 进度 | KPI、进度条、明细表。 |
| `startDate` | 启动日期 | 项目详情、明细扩展列。 |
| `dueDate` | 计划完成日期 | 延期判断、风险列表、明细表。 |
| `riskLevel` | 风险等级 | 风险页、风险标签、明细表。 |
| `riskNotes` | 风险说明 | 风险详情、项目详情。 |
| `updatedAt` | 更新时间 | 趋势图、顶部同步信息、明细排序。 |
| `isDelayed` | 是否延期 | 延期风险页、KPI、风险标签。 |

后端指标里已有 `summary`、`statusCounts`、`riskCounts`、`ownerLoad`、`provinceCounts`、`businessTypeCounts`、`storeStatusCounts`、`monthlyTrend`、`riskProjects`，所以前端目录和图表应优先使用这些现成数据。

## 推荐桌面布局

```text
左侧固定导航
  logo
  总览
  项目明细
  延期风险
  负责人负载
  区域与店型
  钉钉字段

右侧主工作区
  顶部：页面标题 + 数据来源 + 同步时间 + 总记录数 + 刷新
  筛选：搜索 / 省份 / 业态 / 店态 / 状态 / 风险等级 / 负责人（后续可加）
  KPI：总项目 / 进行中 / 延期项目 / 平均进度 / 高风险
  图表：状态分布 / 风险分布 / 负责人负载 / 更新时间趋势
  表格：项目明细
```

## 左侧导航说明

| 导航 | 对应后端数据 | 页面重点 |
| --- | --- | --- |
| 总览 | `/api/snapshot`、`/api/metrics.summary` | 总记录数、同步时间、总项目、进行中、延期、平均进度。 |
| 项目明细 | `/api/projects.items` | 项目名、省份、业态、店态、状态、负责人、进度、计划完成、风险。 |
| 延期风险 | `/api/metrics.riskProjects`、`isDelayed`、`riskLevel`、`riskNotes` | 延期项目、高风险项目、计划完成日期、负责人。 |
| 负责人负载 | `/api/metrics.ownerLoad`、`owner` | 负责人项目数量、负载排行、可筛选负责人。 |
| 区域与店型 | `provinceCounts`、`businessTypeCounts`、`storeStatusCounts` | 省份分布、业态分布、店态分布。 |
| 钉钉字段 | `/api/projects.fieldCatalog`、`rawFields` | 钉钉原始字段目录、字段类型、非空数量；用于检查字段可用性。 |

`钉钉字段` 比 `资料/模板` 更适合现在的系统阶段，因为后端已经有 `fieldCatalog` 设计，能帮助判断真实钉钉表里哪些字段可展示、可筛选、可扩展。

## 组件建议

| 组件 | 建议 |
| --- | --- |
| 主按钮 | 深绿底白字，文字直接，如“刷新”。同步如果需要密钥，不建议暴露成普通前端按钮。 |
| 次按钮 | 白底绿字绿边框，用于“重置筛选”“查看详情”。 |
| 图标按钮 | 刷新、筛选展开、表格列设置等用图标，hover 显示 tooltip。 |
| 筛选输入 | 统一 38-40px 高度，focus 用品牌绿描边。 |
| 状态标签 | 小号圆角或轻量 pill，用颜色表达状态，不要太花。 |
| KPI 卡片 | 白底、细边框、顶部 4px 色条，数字大但不过分。 |
| 表格行 | hover 用极浅绿；风险列用标签，不整行变红。 |
| 空状态 | 简短文案 + 可选线性图标，不做插画。 |

## 状态标签建议

当前真实缓存中的 `status` 主要是“紧急 / 一般 / 未分类”，`riskLevel` 当前多为“低”，所以视觉上不要假设已有复杂状态流。建议先按现有字段轻量表达：

| 类型 | 背景 | 文字 |
| --- | --- | --- |
| 紧急 | `#F8D8CF` | `#C2412D` |
| 一般 | `#F4E8D4` | `#B7791F` |
| 未分类 | `#EEEAE4` | `#6F6A61` |
| 延期 | `#F8D8CF` | `#C2412D` |
| 低风险 | `#DCEBE2` | `#317754` |
| 中风险 | `#F4E8D4` | `#B7791F` |
| 高风险 | `#F8D8CF` | `#C2412D` |

如果后续钉钉表把项目状态拆成更完整的节点（如施工图、待采购、摆场、闭环等），再扩展状态色，不要现在先设计一套不存在的复杂流程。

## 表格建议

默认列建议：

```text
项目名称 / 省份 / 业态 / 店态 / 状态 / 负责人 / 进度 / 计划完成 / 风险
```

可扩展列建议：

```text
启动日期 / 更新时间 / 风险说明 / 钉钉原始字段
```

表格默认不展示所有钉钉原始字段，避免横向过宽。更好的做法是在项目详情抽屉里展示 `rawFields`，或在“钉钉字段”页里管理哪些字段进入表格。

## 细节建议

- 侧边栏宽度保持 220-240px，不要为了展示 logo 做得太宽。
- 卡片圆角控制在 6-8px，别做过度圆润的消费 App 感。
- 表格行高 44-52px，适合项目管理场景。
- 图表不要全绿，保留必要区分；但主视觉锚点始终是绿。
- 页面背景用暖白 `#F7F3EC`，卡片用白色，边框用 `#DDD4C8`。
- 重要数字不要用太多颜色，颜色留给状态。
- 文案优先业务语言：项目、节点、风险、负责人、更新时间、资料状态。

## 后续小模块

| 模块 | 数据基础 | 说明 |
| --- | --- | --- |
| 项目详情抽屉 | `projects.items`、`rawFields` | 展示节点、风险说明、原始钉钉字段。 |
| 字段目录页 | `fieldCatalog` | 查看字段类型、非空数量，辅助决定表格列。 |
| 风险复盘 | `isDelayed`、`riskLevel`、`riskNotes` | 记录延期、返工、资料缺失等原因标签。 |
| 区域/店型分析页 | `provinceCounts`、`businessTypeCounts`、`storeStatusCounts` | 给空间视觉部门看不同区域和店型的项目分布。 |

## 一句话原则

前端可以有源氏木语的颜色和气质，但目录和界面结构必须跟钉钉后端真实字段走。
