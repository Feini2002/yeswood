负责人身份

同名不同责任身份分开统计；显示名尽量贴近日常叫法，统计按 responsibility slot 归集。

阶段与进度

当前阶段与下一提醒统一读取阶段提醒口径表；下游事实可推动阶段向后，字段漏填只作为数据缺口。采购时间只表示采购中，采购完成情况已完成才进入待摆场；摆场开始时间填入即进入摆场中，摆场文件发出即默认摆场结束。

筛选参数

省区、店态、业务类型、阶段等筛选作用于项目情况与部分下钻；同页 hash 变化不触发全量重载。

规则来源与同步

把项目阶段口径、Deadline、提醒卡点和延期判定沉淀成统一规则，由后台硬规则驱动提醒与记录。可执行规则在代码与 JSON；人类可读在 docs/rules/operational-rulebook.md；运营摘要在本开发文档规则章节。不手写完整矩阵副本。

  • 可执行规则 - src/backend/hardDecorationDeadlineRules.mjs 与 data/rules/。
  • 人类可读规则 - docs/rules/operational-rulebook.md。
  • 运营展示 - 本页摘要、状态、关键口径。

延期提醒规则

硬装 Deadline 仍按复尺时间 Y 和中国工作日矩阵生成；软装、采购、摆场先复用统一阶段提醒表输出下一动作，不把单个漏填字段当作主提醒。

  • 复尺时间 Y - 进入平面方案启动约束。
  • 阶段提醒表 - 统一输出待采购、待摆场、等待摆场结束、项目待闭环等动作。
  • 数据缺口 - 上游字段漏填单独提示，不阻断已发生的下游阶段。

负责人责任身份与数据通道

负责人自然人、责任专业和业务范围分开处理；外部来源姓名保留原样，指标和选择器使用本地责任身份 id。

待核对通道

多身份负责人仅命中负责人总列时，不进入明确主指标，进入待核对通道。

安全边界

本地切分只发生在责任身份解析、指标过滤和前端展示层，不回写来源表单。

紧急 / 非紧急待处理 Top5

团队工作完成情况模块的两列待处理队列。入选、分组、计划开业目标风险与排序口径与后台 teamWorkCompletionReview.mjs 保持一致。

人工标记紧急

项目状态等字段文本含紧急，且不含不紧急、非紧急、一般。

非紧急队列

含一般、不紧急、非紧急，或未填写优先级字段的项目；即使逾期也留在非紧急列并显示风险。

Top5 排序

按当天计划开业目标风险排序：逾期、今日交付、7天内、14天内、未来、目标待核对；日期待核对靠后，同风险带叠加公司阶段滞后，采购、摆场中、待闭环按中度补强。

口径边界

交付窗口按计划开业目标日计算，只保留为周期说明，不作为 Top5 主排序依据；紧急标记只作分组，不替代计划开业目标风险。

面积与店态 Deadline 矩阵

所有节点都以复尺时间 Y 为基准，Y+N 表示第 N 个中国工作日。法定双休和节假日不计入 Y+N；2000㎡以上暂按1500～2000㎡超体店时间定。

链路规则与判定原则

硬装链路

延期提醒规则

  1. 复尺时间 Y

    只要有复尺时间，项目进入平面方案启动约束，并触发一次尽快启动的提醒。

  2. Y + 1

    复尺日之后第 1 个中国工作日内必须启动平面方案；超出窗口仍未启动，记录超时启动。

  3. Y + X

    X 由面积对应店态决定，作为平面方案截止；截止前设置一次结束提醒。

  4. 施工图终审

    终审超出截止前判定施工图阶段延期完成。

判定原则

规则与责任边界

表单填写优先

表单为空再兜底

数据来源与更新

数据来源与更新

说明外部源表单、同步入库、汇总数据生成和前端取数路径。

源表单同步与存储汇总数据前端取数

源表单来源

项目与人员数据来自外部源表单；表单字段含义与录入校验由业务侧维护，看板侧只读消费。

同步与存储

手动/自动同步将外部源数据写入本地数据库与 snapshot；前端通过 API 读取，不回写源表单。

汇总数据生成

小组页等重接口走 snapshot 签名缓存与预计算 worker，避免每次请求全量重算。

前端如何取数

首屏由 loadDashboard 编排；各页按需拉取 metrics、profile、team 域接口与项目目录。
